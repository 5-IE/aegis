Aegis Device Signing Tester

Generate a P-256 device key in your browser, register it, and produce a signed POST /api/v1/presence request — as ready-to-paste curl or a live call. Fully client-side; nothing is sent anywhere except the requests you fire.

0 · Config

The JWT must belong to a learner. Get it by logging in via POST /auth/login (or copy it from the app / Postman). The device key you register here binds to whatever user that JWT identifies.

1 · Device key

A P-256 key pair is generated with WebCrypto and stored in this browser's localStorage, so it survives reloads — just like the app persists its key in the Keychain. This is the browser's stand-in for the Secure Enclave.
—

2 · Register the device

One-time, JWT-only (no signature). Stores the public key on the user's record. Run this once per user × device.

3 · Sign & send presence

The exact bytes typed here are what get hashed and sent — the tool never re-serializes them, so what you see is what is signed.

Notes & troubleshooting

  • Live fetch blocked? The backend has no CORS configured, so a browser POST with custom headers is blocked by the preflight. Use the Show curl buttons — those always work from a terminal. Live fetch only succeeds if CORS is enabled or you serve this page from the same origin.
  • 403 "No device registered" → run step 2 first (and make sure the JWT is the same user).
  • 403 "Invalid device signature" → body bytes or key mismatch. Re-copy the exact body; regenerate/register the key.
  • 400 invalid_request → missing headers or your clock is off by >60 s from the server.
  • 204 → success. 🎉
